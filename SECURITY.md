# Security Policy

## Supported Versions

Security fixes are applied to the latest published release and the `main` branch.

## Reporting A Vulnerability

Please do not open a public GitHub issue for suspected vulnerabilities.

Instead, report privately to the maintainer:

- Open a private security advisory if available for this repository.
- Or contact the repository owner directly and include:
  - summary of the issue
  - affected versions or commit range
  - reproduction steps or proof of concept
  - impact assessment
  - suggested mitigation (if known)

You can expect an initial acknowledgment within 3 business days.

## Disclosure Guidelines

- Give maintainers reasonable time to investigate and patch.
- Coordinate public disclosure after a fix is available.
- Avoid sharing exploit details publicly before remediation.
